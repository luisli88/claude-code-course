# dev-vibe-starter

A structured 8-week program to become a **vibe programmer** — someone who works with AI (Claude Code) as a co-pilot and ships software through well-crafted specs rather than brute-force coding.

## Goal

By the end of this program you will:

- Use **Claude Code CLI** as your primary coding tool — steering it with intent, not fighting it with commands
- Write **executable specs** that Claude can implement end-to-end
- Navigate and debug code at speed in **Neovim + LazyVim**
- Own your terminal environment (Ghostty, ripgrep, fzf, delta)
- Master Git workflows used on professional teams
- Build production-quality **Go microservices**

## Program structure

| Week | Topic | Key outcome |
|------|-------|-------------|
| 0 | Setup | Reproducible dev environment running |
| 1 | Claude Code CLI | Ship a feature using only Claude + specs |
| 2 | Neovim + LazyVim | Navigate a codebase without a mouse |
| 3 | Debugging | Step-debug Go code with nvim-dap |
| 4 | Terminal mastery | Find anything in any codebase in seconds |
| 5 | Advanced Git | Rewrite history cleanly, resolve conflicts |
| 6 | Go backend | Microservice with >80% test coverage |
| 7 | Spec Driven Development | Write specs Claude executes perfectly |
| 8 | Agent workflows | Claude as a controlled autonomous agent |

## How to use this repo

1. Read `roadmap/WEEK-0-setup.md` and run the install script for your OS.
2. Work through one week at a time. Each week has objectives, exercises, a project, and a pass/fail evaluation gate.
3. Use `evaluation/self-checklist.md` weekly to confirm you're ready to advance.
4. Track your skill growth in `evaluation/skill-matrix.md`.

## Quick start

```bash
# macOS
./install-macos.sh

# Arch Linux
./install-arch.sh
```

## Philosophy

> Write specs, not code. Steer Claude, don't fight it.

The best vibe programmers spend their time on *what* to build and *how to verify it*, then let Claude handle the implementation. Spec Driven Development (SDD) is the methodology that makes this reliable.
