## Rating scale

### Terminal (1–5)

1. I use `cd`, `ls`, `cat` to get around
2. I can search with `grep` and find files with `find`
3. I use `rg` and `fzf` confidently, compose pipes
4. I have custom aliases, use `bat`/`delta`, live in the terminal
5. I can navigate any unfamiliar codebase in under a minute using terminal tools only

### Git (1–5)

1. I know `add`, `commit`, `push`, `pull`
2. I can branch, merge, and resolve simple conflicts
3. I use interactive rebase, `git add -p`, and `git bisect`
4. My commit history is always clean and readable before PRs
5. I can rewrite complex history confidently and onboard others on Git workflows

### Go (1–5)

1. I understand Go syntax and can read Go code
2. I can write simple HTTP handlers and run tests
3. I write table-driven tests and know stdlib patterns
4. My code has >80% coverage and uses structured logging
5. I design Go services with clear package boundaries and can review others' Go code

### Neovim (1–5)

1. I can open and edit files, save and quit
2. I use normal mode, search with `/`, and navigate with `hjkl`
3. I use LSP (`gd`, `gr`, `K`), split windows, and fuzzy search
4. I navigate large codebases without a mouse, use `<leader>` shortcuts fluently
5. I customize my config and debug issues in Neovim itself

### SDD — Spec Driven Development (1–5)

1. I understand what a spec is
2. I can write a basic GIVEN/WHEN/THEN scenario
3. I write specs with acceptance criteria that map to tests
4. Claude implements my specs correctly on the first or second attempt
5. I can spec an entire feature milestone and guide Claude through multi-step implementation

### Claude Code — Vibe Programming (1–5)

1. I've used Claude to generate code snippets
2. I use Claude to implement features and review diffs
3. I steer Claude with specs and constraints, rarely edit output manually
4. I use Claude for multi-step tasks with checkpoints and clear scope
5. I design workflows where Claude does 80%+ of implementation work and I validate quality
